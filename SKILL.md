# A股盘前短线标的筛选 v6.9.33

基于前一日完整收盘数据筛选当日有望上涨的A股短线标的。**不追高是硬纪律。**

## 一、步骤零：北京时间获取（最高优先级）

仅通过公共网络授时 API 获取精确北京时间，不依赖本地系统时钟。3源备用：`timeapi.io` → `worldtimeapi.org` → `worldclockapi.com`，全部不可达→直接中止。所有搜索 query 使用 `data_date`，输出文件名使用 `prediction_date`。

> 📄 完整代码见 `references/step0-beijing-time.py`

## 二、可配置参数

从 `/workspace/策略调整记录.json` 数组末条 `params` 字段读取，共11项参数，默认值：`search_budget=25, northbound_threshold=3000, consecutive_weeks=2, win_rate_drop_threshold=10, limit_down_threshold=100, max_adjust_params=3, confidence_position_enabled=true, strategy_concentration_pct=30, data_tier_l2_skip_on_unavailable=true, data_tier_l3_downgrade_to_signal=true, strategy_a_weak_market="closed"`。

## 三、系统告警与文件容错

告警触发场景含30+种（ERROR/WARNING/INFO），详情见 `references/log-alert.py`。所有文件读写使用 safe_ 系列函数，JSON 追加必须用 `safe_append_json`。

> 📄 完整代码见 `references/safe-file-ops.py`

## 四、前置检查（步骤1-8）

**1.节假日**：搜索中国股市交易日历→data_date 或 prediction_date 为节假日→跳过；长休≥3日→弱市+仓位≤30%+搜索预算+5

**2.极端行情**：上证跌≥3%→跳过；涨≥3%→仓位30%极端上涨降仓防追高；跌停>threshold→跳过

**3.外围市场**：美股三大指数均跌>2%→弱市仓位≤30%；恒生跌>3%→弱市仅超跌反弹；人民币波动>0.5%→暂停策略E。美股/港股假期→跳过此检查

**3A.大盘代理**：深证成指（sz399001）跌>1%→仓位降一档（下限20%）。数据不可得→跳过

**4.持仓行情同步**：遍历推荐历史中 `type="holding"` 记录，搜索当日收盘价更新 current/pnl_pct/market_value/pnl_amount。搜不到→log_alert WARNING 保留旧数据。

> 📄 完整代码见 `references/pre-check-holdings-sync.py`

**4A.做T评估**：详见「做T评估」章节。输出 `type="do_T_eval"` 追加到推荐历史。

**4B.持仓跟踪.xlsx同步**：将 holding 收盘价同步写入 `/workspace/持仓跟踪.xlsx` 的「持仓明细」sheet。仅更新当前价/市值/盈亏/更新日期，不修改成本/持仓量。代码匹配时4位自动 `zfill(6)` 补齐。

**4C.持仓危机检查**：检查跌停（<-9.5%）、浮亏>15%、触发硬排除规则，不阻断筛选但对话置顶告警。

> 📄 完整代码见 `references/pre-check-crisis.py`

**5.推荐历史清理**：逐日期文件独立清理，保留7日内 recommendation + 90日内 holding + 所有 weekly_review/strategy_check/do_T_eval/do_T。

**6.文件初始化**：策略调整记录.json取末条version+params，损坏→默认BUILTIN_VERSION。首次运行或版本变更→追加type="strategy_check"记录。

> 📄 完整代码见 `references/pre-check-version.py`

**7.财报季**：1/3/4/8/10月→事件驱动权重×1.5+仓位+5%，动量延续涨幅上限7%→8%

**8.大盘环境**：通过腾讯API获取三大指数实时行情，结合pytdx均线判断MA20位置。

| 环境 | 条件 | 总仓位 | 动量(A) | 超跌(B) | 事件(C) | 回调(D) | 资金(E) | 北向(F) |
|------|------|--------|---------|---------|---------|---------|---------|---------|
| 强市 | 上证>MA20且MA5>MA10>MA20且涨跌比>2:1且成交>20日均×1.2 | 70-80% | 35-40% | 10-12% | 10-12% | 10-12% | 5-8% | 3-5% |
| 震荡 | 上证在MA20±2%或涨跌比≈1:1 | 50-60% | 12-17% | 10-13% | 8-10% | 12-15% | 5-8% | 3-5% |
| 弱市 | 上证<MA20×0.98或涨跌比<1:2或成交<20日均×0.8 | 30-40% | **0%** | 12-15% | 5-8% | 8-12% | 3-5% | 3-5% |

边界：上证在MA20×0.98~MA20但不满足震荡→弱市。三项指标矛盾时→保守取弱一档。

## 五、数据采集（步骤10A-10G）

### 5.1 行情采集（步骤10A）：三级降级

| 优先级 | 数据源 | 方式 | 覆盖字段 | 失败处理 |
|--------|--------|------|---------|----------|
| **Tier 1** | 腾讯财经 qt.gtimg.cn | HTTP API 批量（40只/批） | OHLCV + 量比/换手率/振幅/PE/总市值 | → 降级 Tier 2 |
| **Tier 2** | 新浪财经 hq.sinajs.cn | HTTP 批量（80只/批） | 仅 OHLCV | → 降级 Tier 3 |
| **Tier 3** | 通达信 pytdx | Python 库直连行情主站 | OHLCV + 量比/换手率 + MA/KDJ/RSI/MACD | → log_alert ERROR 中止 |

**腾讯API字段索引**：`[3]`=最新价 `[5]`=开盘 `[33]`=最高 `[34]`=最低 `[37]`=成交额(万元) `[38]`=换手率 `[39]`=PE(TTM) `[43]`=振幅 `[44]`=总市值(亿元) `[49]`=量比

**新浪降级代理**：量比→成交额分位数P75 | 换手率→成交额/前收盘/100 | 主力净流入→跳过

**pytdx 连接**：`pip install pytdx --break-system-packages`，主站 `119.147.212.81:7709`，备用 `120.76.152.87`/`47.92.127.118`/`59.173.18.140`

> 📄 完整代码见 `references/fetch-all-stocks.py`

### 5.2 行业补全（步骤10B）：双优先级查表

**优先级1**：硬编码精准覆盖（84只，见 `ashare_screener.py` 中 `HARDCODED_INDUSTRY` 字典）。**优先级2**：210+代码段查表（`INDUSTRY_MAP`），覆盖600000-609999/000001-005999/300000-304999全部A股。两表均未命中→"未知"（v6.6.29已实现零覆盖率，不应出现）。

### 5.3 历史K线（步骤10C）：pytdx 28指标

pytdx `get_security_bars()` 获取60日K线，计算MA5/MA10/MA20、MACD、RSI(14)、KDJ、布林带、WR、OBV、DMI、涨停标记、上市天数。不可达时 K线数据为空字典，依赖K线的策略(I-Q)和信号过滤(9-16)自动跳过。

### 5.4 财务数据（步骤10D+10E）

**step10D（已废弃降级）**：东方财富datacenter-web API（质押/商誉/解禁）全部废弃，返回空字典。**step10E（F10单股API）**：`ZYZBAjaxNew` 逐只拉取ROE/净利润/营收/EPS，用于信号过滤#21净利润亏损排除。

### 5.5 风险事件（步骤10F）：解禁/可转债/业绩预告

限售解禁数据源：qqjjsj.com 结构化解析（内置38只缓冲降级）。可转债：内置已知到期/强赎列表。业绩预告：7月1-15日主板强制披露窗口。

### 5.6 拥挤度（步骤10G）：机构持仓+融资过热

机构持仓：F10 ShareholderResearch API 拉取基金持仓占比+前十大股东机构减持数量。融资过热代理：换手率>20%且量比>2.5→代理融资买入占比>25%。

### 5.7 主营业务（步骤10H）：CoreConception API

F10 CoreConception API 拉取 `hxtc` 核心题材，三级策略提取主营业务描述（≤30字）：①匹配含"主营/业务/研发/生产/销售"关键词项 → ②匹配以公司名开头的内容 → ③兜底取第2项。自动清除"公司主要从事"等冗余前缀。MD/HTML输出表新增"主营业务"列，推荐历史JSON同步记录。

### 5.8 数据源监控

每次筛选记录至 `/workspace/数据源监控.json`。腾讯一级连续10次不可达→CRITICAL告警。恢复后自动归零。

## 八、行业集中度

同申万一级行业：弱市≤4只（强/震荡市≤3只）。同策略集中度≤30%。**弹性规则**：弱市下第N+1只若 `_tie_score ≥ 第N只 × 0.90` 则保留。

## 九、推荐不足降级

3只→全部+放宽至中 | 2只→全部+仅≥中置信 | 1只→仅高置信 | 0只→"无合适标的"+空标记

## 十、十七策略（ABCDEFGHIJKLMNOPQ）

**A动量延续(优1·权重10%)**：涨3-7%+量比1.5-3.0+弱市关闭。进场：ATR×pos×0.4×量比修正。仓位：强35-40%/震荡12-17%/弱关闭。

**B超跌反弹(优2·权重10%)**：跌2.5-9.5%+振幅>3%+反弹确认(close>low×1.01)。深度跌幅加分(<-7%:+2, <-5%:+1)。进场：低点上方30%分位。止损-7%+止盈+7%。

**C事件驱动(优3·权重8%)**：涨1-2%+量比≥1.0或财报季。弱市关闭。事件时效5级衰减：当日100%→次日80%→第3日50%→4-7日30%→>7日10%。

**D回调企稳突破(优4·权重7%)**：涨3-6%(弱市上限7%)+振幅1.5-10%+阳线。D与A冲突→A优先，D与B冲突→D优先。振幅四档(1.5-3%:-1, 3-6%:+1, 6-8%:0, >8%:-1)。进场：低点上方15%支撑区+40%分位。

**E资金埋伏(优5·权重6%)**：涨0-1%+主力流入>3000万。代理兜底：量比≥0.6+换手≥0.5%+阳线。进场：振幅下25%分位。

**F北向资金(优6·权重5%)**：涨0-1%+主力流入>5000万+近5日持续≥3日。E→F升级。代理兜底：量比≥0.8+换手≥1.5%+持续≥2日。进场：`low+(close-low)×0.3`。

**G横盘突破(优7·权重4%)**：涨1.0-3.0%+阳线+振幅1.5-6%+量比≥1.0。弱市低吸不追。cs中心2.0±1.5。

**H地量见底(优8·权重3%)**：涨-3~1.0%+阳线+量比<1.0+(锤子线或十字星)。进场：`low+(close-low)×0.4`。

**I-Q 形态策略**（弱市跳过，仅强/震荡市匹配）：I均线粘合突破 | J龙回头 | K缺口回补 | L黄金坑 | M涨停回调 | N新高突破 | O回踩均线 | P地量反弹 | QW底突破。详细条件见代码。

**评分公式**（v6.9.25）：
```
_tie_score = (vs×0.25 if D else vs×0.30) + (ts×0.35 if D else ts×0.30) + cs×0.30 + (1.0-so/10)×0.10 + sector_bonus + ma_bonus
score = base×2 + round(_tie_score×8)
```
穿插策略权重：A(10%)→B(9%)→C(8%)→D(7%)→E(6%)→F(5%)→G(4%)→H(3%)→I(2%)→J(1%)→K(0%)→L(-1%)→M(-2%)→N(-3%)→O(-4%)→P(-5%)→Q(-6%)。`_tie_score` 由 `max(0, _tie_score)` 保护。

**二次评估（打破平局）**：score → 策略优先级 → tie_score → 量比 → 换手率偏差 → 板块热度。

**置信度-仓位联动**：★★★→仓位上限 | ★★→仓位中值 | ★→仓位下限。受 `confidence_position_enabled` 开关控制。

## 十一、新闻筛查

排除：减持/暴雷/立案/诉讼/下调评级 | 观察：异常波动/解禁/高管减持 | 加分：预增/合同/调研/上调评级。时效衰减：当日100%→2-3日70%→4-7日30%→>7日0%。

## 十二、做T评估（仅对持仓）

浮亏<5%或浮盈→观望 | 5-10%→≤总持仓1/3 | 10-15%→≤1/4 | >15%→不做T。评估：止跌信号+波动≥3%+非放量跌+板块预期+无利空。目标2-3%止盈，-3%止损。累计成功≥3次→重置计数器；连续2次失败→放弃。

## 十三、Markdown输出（含筛选概况）

输出文件：`/workspace/短线标的_YYYYMMDD.md`。报告头部 + 筛选管道 + 推荐标的表格(14列) + 策略分布统计 + 策略说明 + 硬排除TOP5 + 免责声明。

**对话中必须输出**：📊 筛选概况（6级漏斗数量→策略分布→排除TOP5）。阶段通过数必须与MD表格行数一致。
