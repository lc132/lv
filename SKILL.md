# A股盘前短线标的筛选 v6.9.10

基于前一日完整收盘数据筛选当日有望上涨的A股短线标的。**不追高是硬纪律。**

## 步骤零、北京时间获取（最高优先级，必须第一步执行）

**核心原则**：仅通过公共网络授时 API 获取精确北京时间。不依赖本地系统时钟、不降级到 zoneinfo/pytz。

> 📄 完整代码见 `references/step0-beijing-time.py`

**v6.6.45 新增3源备用**：`timeapi.io` → `worldtimeapi.org` → `worldclockapi.com`，兼容不同 JSON 字段名（`dateTime`/`datetime`/`currentDateTime`），自动清理时区后缀。全部不可达→直接中止。

所有搜索 query 使用 `data_date`，输出文件名 `/workspace/短线标的_YYYYMMDD.md` 使用 `prediction_date`。

## 可配置参数

从 `/workspace/策略调整记录.json` 数组末条 `params` 字段读取，共11项参数，默认值：`search_budget=25, northbound_threshold=3000, consecutive_weeks=2, win_rate_drop_threshold=10, limit_down_threshold=100, max_adjust_params=3, confidence_position_enabled=true, strategy_concentration_pct=30, data_tier_l2_skip_on_unavailable=true, data_tier_l3_downgrade_to_signal=true, strategy_a_weak_market="closed"`。

参数用途说明：`search_budget`步骤10搜索次数 | `northbound_threshold`策略E主力资金流入阈值(万元)，默认3000万 | `consecutive_weeks`预留(周线趋势连续周数，当前未启用) | `win_rate_drop_threshold`预留(胜率回撤触发%，当前未启用) | `limit_down_threshold`步骤2跌停阈值 | `max_adjust_params`仅周六Task3回滚参数修改上限，每日筛选不执行回滚 | `confidence_position_enabled`置信度-仓位联动开关(true=启用) | `strategy_concentration_pct`步骤17同策略上限(%) | `data_tier_l2_skip_on_unavailable`数据不可达跳过 | `data_tier_l3_downgrade_to_signal`L3降级开关 | `strategy_a_weak_market`预留(弱市策略A开关，当前始终关闭)。v6.9.4已删除：max_holding_days/circuit_breaker_threshold_pct/do_t_success_reset_count/conversion_rate_*（step9A/B/C持仓风控未实现）

## 前置检查（步骤1-8）

**1.节假日**：搜索中国股市交易日历→data_date 或 prediction_date 为节假日→跳过；长休≥3日→弱市+仓位≤30%+搜索预算+5

**2.极端行情**：上证跌≥3%→跳过；涨≥3%→仓位30%极端上涨降仓防追高；跌停>threshold→跳过

**3.外围市场**：美股三大指数均跌>2%→弱市仓位≤30%；恒生跌>3%→弱市仅超跌反弹；人民币波动>0.5%→暂停策略E。美股/港股假期→跳过此检查

**3A.大盘代理**：盘前检查深证成指（sz399001）作为大盘强弱代理指标。深成指跌>1%→仓位降一档（强→震荡→弱，下限20%）。若数据不可得→跳过此检查。

**4.持仓行情同步**：遍历推荐历史中 `type="holding"` 记录，先暂存 `old_current`，再搜索当日收盘价→仅在API返回成功后将 `old_current` 写入 `prev_close`→更新current/pnl_pct/update_date，同时计算 market_value=round(current×shares,2) 和 pnl_amount=round((current-cost)×shares,2)。搜不到→log_alert WARNING保留旧数据，prev_close 回退为旧值。`safe_write_json` 写回推荐历史。prev_close 用于步骤4C跌停检测。

**4A.做T评估**：对持仓进行做T可行性评估，详见「九、做T评估」。输出 `type="do_T_eval"` 追加到推荐历史，回溯检查昨日 do_T_eval 的 `do_T_feasible` 字段。

**4B.持仓跟踪.xlsx同步**：步骤4完成后，将更新后的 holding 收盘价同步写入 `/workspace/持仓跟踪.xlsx` 的「持仓明细」sheet。
- 仅更新「当前价」列（列8）和「市值」列（列9）、「盈亏额」列（列10）、「盈亏率」列（列11）、「更新日期」列（列12）
- 按 code 匹配行，不新增/删除行，不修改成本/持仓量等字段
- 代码匹配时自动处理Excel数字化的代码：`str()` 后检测长度，4位自动 `zfill(6)` 补齐
- 若 xlsx 不存在或结构异常→log_alert WARNING，跳过；若某 code 在 xlsx 中找不到→log_alert WARNING
- 同步后 save

> 📄 完整代码见 `references/pre-check-holdings-sync.py`

**4C.持仓危机检查**：步骤4持仓行情更新后，遍历 holding 记录检查危机信号。不阻断筛选流程，但在对话输出中置于筛选概况上方优先展示。

## 二、信号质量过滤（17项，v6.9.4）

1. **假动量**：高开>3%且收<开×0.98→排除
2. **诱多**：盘中最高涨>5%且收<开×1.01→排除
3. **缩量涨停**：涨幅>5%+量比<0.5→排除
4. **振幅过大**：振幅>15%→排除
5. **跌停板异动**：chg<-9%+振幅>12%→排除
6. **缩量下跌**：chg<-3%+量比<0.3→排除
7. **高换手低涨幅**：换手>20%+涨跌幅<2%→排除
8. **首阴标记**：-3%<chg<0+换手>3%→标记`_first_yin`(不排除，仅加分)
9. **均线空头排列**：MA5<MA10<MA20→排除
10. **MACD顶背离**：价格新高但DIF未新高→排除
11. **RSI超买**：RSI(14)>80→排除
12. **缩量反弹**：连续3日量能递减+当日反弹>2%+量比<0.6→排除
13. **KDJ高位死叉**：J值>100后下穿K/D线→排除
14. **涨停次日高开低走**：前日涨停+当日高开低走收阴→排除
15. **布林带突破失败**：带宽<5%+放量(量比≥1.5)但收阴→排除
16. **20日涨幅>45%**：防止追高爆炒股→排除
17. **净利润亏损**：fundamental_data中净利润为负→排除

## 三、行业集中度

同申万一级行业≤3只，评分前先行业预分配确保≥2个行业。**同策略集中度**：总推荐中同策略标的≤{strategy_concentration_pct}%（如5只中同策略≤{strategy_concentration_pct*5//100}只，由代码预计算），超出则降分排序取前。

## 四、推荐不足降级

3只→全部+放宽至中 | 2只→全部+仅≥中置信 | 1只→仅高置信 | 0只→"无合适标的"+追加空标记

**J 龙回头（v6.9.0 新增）**：20日涨幅>5%（强势股）+回调10-20%+缩量(量比<0.5)收阳；逻辑：强势股回调至支撑位，缩量企稳后有望反弹；进场价：`low+(close-low)*0.35`低吸；仓位：震荡8-10%/弱市5-8%。

**K 缺口回补（v6.9.1 新增）**：前日跳空高开1-5%+当日回踩缺口上沿(low≤前高×1.01)+收阳；逻辑：跳空缺口确认支撑，回踩不破则继续上行；进场价：`close*1.005`追涨；仓位：震荡8-10%/弱市5-8%。

**L 黄金坑（v6.9.1 新增）**：5日内急跌≥8%+V型反弹≥4%+放量(量比≥1.2或换手≥3%)收阳；逻辑：急跌后快速反弹为黄金坑形态，主力洗盘后拉升；进场价：`low+(close-low)*0.3`低吸；仓位：震荡8-10%/弱市5-8%。

**M 涨停回调（v6.9.2 新增）**：近5日涨停+回调5-15%（至涨停价50-62%位置）+缩量(量比<0.7)收阳；逻辑：涨停后回调至支撑位，缩量企稳可博第二次拉升；进场价：`close*0.998`平吸；仓位：震荡6-8%/弱市3-5%。

**N 新高突破（v6.9.2 新增）**：收盘价=20日新高+放量(量比≥1.5)阳线；逻辑：突破前高压力位，放量确认有效突破；进场价：`close*1.01`追涨；仓位：震荡8-10%/弱市5-8%。

**O 回踩均线（v6.9.2 新增）**：60日涨幅>20%+回踩MA20±1%+缩量(量比<0.7)收阳；逻辑：强势股回踩MA20支撑，缩量确认支撑有效；进场价：`close*1.002`平吸；仓位：震荡8-10%/弱市5-8%。

**P 地量反弹（v6.9.3 新增）**：连续3日缩量至地量+当日放量≥2x+涨2-5%阳线；逻辑：地量见地价后放量反弹，确认底部反转；进场价：`close*1.005`追涨；仓位：震荡6-8%/弱市3-5%。

**Q W底突破（v6.9.3 新增）**：20日内两底相差<5%+放量(量比≥1.5)突破颈线(两底间最高点)1%+阳线；逻辑：经典W底形态，突破颈线确认反转；进场价：`close*1.01`追涨；仓位：震荡8-10%/弱市5-8%。

**权重公式**（`_tie_score`策略权重分量）：`(1.0 - so/10) × 0.10`，其中 so 为策略优先级（A=0, B=1, C=2, D=3, E=4, F=5, G=6, H=7, I=8, J=9, K=10, L=11, M=12, N=13, O=14, P=15, Q=16）。权重递减：A(10%)→B(9%)→C(8%)→D(7%)→E(6%)→F(5%)→G(4%)→H(3%)→I(2%)→J(1%)→K(0%)→L(-1%)→M(-2%)→N(-3%)→O(-4%)→P(-5%)→Q(-6%)。`_tie_score` 完整公式：`vs×0.30 + ts×0.30 + cs×0.30 + 策略权重 + sector_bonus + ma_bonus`。
注：so>=10的策略权重为负值，实际执行中 `_tie_score` 由 `max(0, _tie_score)` 保护，确保不会因策略权重而导致负分。

## 六.策略进场价计算规则（v6.6.31 重构：基于ATR历史数据推算）

**核心原则**：进场价基于当日历史数据（ATR真实波动率/振幅位置/开盘缺口/量比）综合推算，而非简单使用收盘价×固定倍数。

```python
def calc_entry_price(c):
    """基于历史数据推算次日合理进场价，综合考虑ATR/振幅位置/缺口/量比"""
    strategy = c.get('strategy', 'Z')
    close = c.get('close', 0); op = c.get('open', 0)
    high = c.get('high', 0); low = c.get('low', 0)
    prev = c.get('prev_close', close); chg = c.get('change_pct', 0)
    amp = c.get('amplitude', 0) or 0; vol_ratio = c.get('volume_ratio', 1) or 1

    # 当日真实波动(ATR日) — 核心历史数据指标
    if high > 0 and low > 0 and prev > 0:
        tr = max(high - low, abs(high - prev), abs(low - prev))
        atr_pct = tr / prev  # ATR百分比
    else:
        atr_pct = max(amp / 100, 0.015) if amp else 0.02

    # 收盘在振幅位置(0=最低, 1=最高)
    pos = (close - low) / (high - low) if high > low and high > 0 else 0.5
    # 缺口
    gap = (op - prev) / prev if prev > 0 else 0
    # 量比修正
    vol_adj = min(vol_ratio / 1.5, 1.5) if vol_ratio > 0 else 1.0

    if strategy == 'A':   # 动量延续
        if pos > 0.65:      # 强势收盘→高开0.5-1.2%
            gap_e = pos * atr_pct * 0.4 * vol_adj
            return round(close * (1 + max(gap_e, 0.005)), 2)
        elif pos > 0.35:    # 中性→平开/小高开
            return round(close * (1 + atr_pct * 0.25), 2)
        else:               # 尾盘回落→收盘价附近
            return round(close * (1 + gap * 0.5), 2)

    elif strategy == 'B':  # 超跌反弹
        if low > 0 and close > low and pos > 0.3:
            return round(low + (close - low) * 0.3, 2)  # 低点上方30%
        elif chg < -5:
            return round(close * (1 - atr_pct * 0.3), 2)
        else:
            return round(close * 0.995, 2)

    elif strategy == 'C':  # 事件驱动
        return round(close * (1 + max(atr_pct * 0.35 * vol_adj, 0.005)), 2)

    elif strategy == 'D':  # 回调企稳
        if low > 0 and high > low:
            support = low + (high - low) * 0.15
            return round(support + (close - support) * 0.4, 2)
        return round(close * 1.01, 2)

    elif strategy == 'E':  # 资金埋伏
        if high > low and low > 0:
            return round(low + (high - low) * 0.25, 2)  # 振幅下25%分位
        return round(close * 0.995, 2)

    return round(close, 2)
```

**各策略进场价逻辑**：
| 策略 | 进场价逻辑 | 数据依据 |
|------|----------|----------|
| A 动量 | 强势(pos>0.65)→ATR×pos×0.4×量比修正；中性→ATR×0.25；弱势→收盘价附近 | 历史振幅位置+ATR波动率+量比 |
| B 超跌 | 尾盘回升→低点上方30%分位；深度超跌→ATR×0.3下修；其他→收盘价-0.5% | 日内低点支撑+ATR波动率 |
| C 事件 | close×(1+ATR×0.35×量比修正)，最少+0.5% | ATR波动率+量比修正 |
| D 回调 | 低点上方15%为支撑区，在支撑和收盘间40%位置进场 | 日内支撑位判断 |
| E 资金 | 振幅下25%分位挂单，低吸博反弹 | 日内振幅区间下沿 |
| F 北向 | `low + (close-low)*0.3` 或 `close*0.995`，低吸博资金持续流入 | 日内振幅区间下沿 |

## 七、评分公式

**计分规则**（v6.9.4 简化）：`_tie_score = vs×0.30 + ts×0.30 + cs×0.30 + (1.0-so/10)×0.10 + sector_bonus + ma_bonus`。

**v6.6.45扣分**：A策略假突破(`_fake_breakout`)→减3分；C策略量价不足(`_c_downgrade`)→减2分。
置信：≥9★★★ | 6-8★★ | <6★。分值为整数，如遇小数→向上取整判定置信度（如5.2→★★，8.8→★★★）。策略冲突按优先级归类，动量+超跌同时→以动量为准。**v6.7.0**新增G横盘突破(优7)、H地量见底(优8)。E与F互补→E先匹配，F为E的升级。

**置信度-仓位联动**（受 `confidence_position_enabled` 开关控制）：`confidence_position_enabled=true` 时→同一策略内 ★★★→取仓位上限 | ★★→取仓位中值 | ★→取仓位下限。`confidence_position_enabled=false` 时→统一取仓位中值。仓位上限/下限按策略定义取整。

**评分相同时的二次评估（打破平局）**：当多个标的综合评分相同时，按以下优先级依次比较，确定最终排序。**v6.7.4 已实现为 5 元组排序键**（`-score → strategy_priority → -tie_score → -volume_ratio → to_penalty`），`_tie_score` 相同时继续按量比和换手率打破平局：

| 优先级 | 维度 | 规则 | 依据 |
|--------|------|------|------|
| 1 | 策略优先级 | A(动量) > B(超跌) > C(事件) > D(回调) > E(资金) > F(北向) | 策略本身的预期收益率降序 |
| 2 | 量比 | 高者优先 | 量比越高表示当日资金关注度越高，短期爆发力越强 |
| 3 | 换手率 | 中等优先（5%-15%最佳），过低(<2%)排末，过高(>25%非次新)降级 | 过低无人气，过高有出货嫌疑，适中换手率表示健康交投 |
| 4 | 涨跌幅 | 按策略区分：A/E类涨幅低者优先（空间更大），B类跌幅深者优先（超跌更充分），C/D类涨幅适中优先 | 不同策略对涨跌幅的偏好不同 |
| 5 | 板块热度 | 当日资金流入TOP3板块的标的优先 | 板块共振提升上涨概率 |
